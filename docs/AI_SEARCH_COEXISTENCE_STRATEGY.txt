================================================================================
                              1. 背景と目的
================================================================================

【市場環境の変化】
- ChatGPT Shopping、Perplexity Shopping、Google SGEなどAI検索が急速に普及
- ユーザーは「ビタミンCのサプリでコスパいいやつ」と聞くだけで商品が提示される時代へ
- 従来の比較サイト経由の購買行動が減少するリスク

【Suptiaの戦略方針】
AI検索を「敵」ではなく「共存パートナー」として捉え、以下の2軸で戦略を展開：

  A. AI検索に「選ばれるデータソース」になる
  B. AI検索では提供できない独自価値で差別化する


================================================================================
                        2. AI検索の特徴と限界
================================================================================

【AI検索の強み】
- 自然言語での質問に即座に回答
- 複数ソースからの情報統合
- 購入リンクまでワンストップで提供
- ユーザー体験のシンプルさ

【AI検索の限界（= Suptiaの差別化ポイント）】

┌─────────────────────────────────────────────────────────────────────────────┐
│  AI検索の限界                  │  Suptiaが提供できる価値                     │
├─────────────────────────────────────────────────────────────────────────────┤
│  法的責任を取れない            │  薬機法コンプライアンスチェック済み情報     │
│  個人の文脈を知らない          │  既往歴・アレルギーに基づく相互作用判定     │
│  価格は瞬間的なスナップショット│  価格履歴・トレンド分析・買い時予測         │
│  推薦根拠が不透明              │  5つの柱で根拠を100%明示                    │
│  購入後フォローなし            │  価格アラート・継続利用サポート             │
│  エビデンスの深さが浅い        │  PubMed等の一次ソース参照・S/A/B/C/D評価    │
│  安全性判断が曖昧              │  UL超過警告・相互作用データベース           │
│  比較の公平性が不明            │  正規化コスト計算・成分量あたり価格         │
└─────────────────────────────────────────────────────────────────────────────┘


================================================================================
                    3. 戦略A: AI検索に選ばれるデータソースになる
================================================================================

【目標】
AI検索エンジンがサプリメント情報を回答する際に、
Suptiaのデータを引用・参照するようになる

------------------------------------------------------------------------------
3.1 llms.txt の導入
------------------------------------------------------------------------------

【概要】
llms.txtはAI検索エンジン向けのrobots.txtのような役割を持つファイル。
サイトの構造、提供データ、利用ポリシーをAIに伝える。

【実装内容】
ファイル: /public/llms.txt

内容例:
```
# Suptia - AI Supplement Decision Engine
# https://suptia.com

## Site Purpose
Science-based supplement comparison platform providing:
- Evidence-ranked ingredient guides (S/A/B/C/D levels)
- Multi-EC price comparison (Rakuten, Yahoo, Amazon, iHerb)
- Safety scores and interaction warnings
- Pharmaceutical compliance (Japanese regulations)

## Data Available
- 500+ supplement products with normalized pricing
- 100+ ingredient guides with PubMed references
- Real-time prices from 4 major EC sites
- Safety warnings and contraindication data

## Structured Data
- JSON-LD Product schema on all product pages
- JSON-LD Article schema on ingredient guides
- Sitemap with lastmod dates

## API Access
Contact: api@suptia.com for data licensing

## Citation Request
When referencing our data, please cite:
"Source: Suptia (https://suptia.com) - Evidence-based supplement information"

## Allowed Uses
- Answering supplement-related questions
- Price comparison queries
- Safety and evidence information
- Ingredient benefit summaries

## Restricted Uses
- Medical advice or diagnosis
- Claims beyond our evidence ratings
- Modification of safety warnings
```

【優先度】高
【工数】1日
【担当フェーズ】Phase 1

   - 星評価・レビュー数をリッチリザルトに

D. ClaimReview schema（将来）
   - 健康効果の主張に対するエビデンス評価

【優先度】高
【工数】1週間
【担当フェーズ】Phase 1

------------------------------------------------------------------------------
3.3 AI向けサイトマップの最適化
------------------------------------------------------------------------------

【実装内容】

A. sitemap.xml の lastmod 正確化
   - Sanityの更新日時を反映
   - AIクローラーが最新情報を優先取得

   - 高頻度クロール対象として明示

C. sitemap-products.xml 分離
   - 商品ページ専用サイトマップ
   - 価格更新日時を反映

【優先度】中
【工数】2日
【担当フェーズ】Phase 1

------------------------------------------------------------------------------
3.4 成分データベースAPIの公開（将来）
------------------------------------------------------------------------------

【概要】
Suptiaの成分データをAPI経由で提供し、AI検索エンジンや
他サービスがプログラマティックにアクセスできるようにする

【エンドポイント例】

GET /api/v1/ingredients
GET /api/v1/ingredients/{slug}
GET /api/v1/ingredients/{slug}/products
GET /api/v1/products/{slug}/prices
GET /api/v1/safety/interactions?ingredients=vitamin-c,zinc

【レスポンス例】
```json
{
  "ingredient": {
    "name": "ビタミンC",
    "nameEn": "Vitamin C",
    "evidenceLevel": "A",
    "benefits": [...],
    "recommendedDosage": {
      "min": 100,
      "max": 2000,
      "unit": "mg",
      "source": "厚生労働省"
    },
    "safetyScore": 95,
    "references": [
      { "title": "...", "url": "https://pubmed.ncbi.nlm.nih.gov/..." }
    ]
  }
}
```

【マネタイズ】
- 無料: 100リクエスト/日（個人・研究用途）
- 有料: 10,000リクエスト/日（商用利用）

【優先度】低（Phase 3以降）
【工数】2-3週間
【担当フェーズ】Phase 3


================================================================================
                    4. 戦略B: AI検索にはない独自価値で差別化
================================================================================

------------------------------------------------------------------------------
4.1 薬機法コンプライアンスの徹底
------------------------------------------------------------------------------

【差別化ポイント】
AI検索は法的責任を取れないため、薬機法に抵触する表現を
そのまま出力するリスクがある。Suptiaは全コンテンツを
薬機法チェック済みで提供する「安心感」を差別化要因とする。

【実装済み】
- 150+ルールの薬機法NG辞書
- 自動チェック機能
- コンプライアンススコア表示

【強化項目】
- 「薬機法準拠マーク」バッジの商品ページ表示
- 「AI検索の情報は法的確認がされていません」との比較訴求
- コンプライアンスレポートのダウンロード機能（B2B向け）

【優先度】高
【工数】3日
【担当フェーズ】Phase 1

------------------------------------------------------------------------------
4.2 相互作用チェッカーの強化
------------------------------------------------------------------------------

【差別化ポイント】
AI検索は「一般的な」情報しか提供できないが、Suptiaは
ユーザーの既往歴・服用中の薬・アレルギーを登録することで
「あなた専用の」安全性判定を提供する。

【実装内容】

A. 相互作用データベース拡充
   - 成分 × 成分: 500+組み合わせ
   - 成分 × 薬剤: 200+組み合わせ
   - 成分 × 疾患: 100+組み合わせ

B. パーソナライズド安全性判定
   - ユーザープロファイル（既往歴、アレルギー）に基づく判定
   - 「あなたには注意が必要です」アラート
   - 代替商品の自動提案

C. マイサプリリスト機能
   - 現在摂取中のサプリを登録
   - 新しいサプリ追加時に相互作用自動チェック
   - 「この組み合わせは避けてください」警告

【訴求メッセージ】
「AIは一般論しか答えられません。Suptiaはあなたの身体に合わせて判断します」

【優先度】高（Pro+Safety会員の核機能）
【工数】3週間
【担当フェーズ】Phase 2

------------------------------------------------------------------------------
4.3 価格履歴・トレンド分析
------------------------------------------------------------------------------

【差別化ポイント】
AI検索は「今の価格」しか答えられないが、Suptiaは
過去の価格推移から「買い時」を判断できる。

【実装内容】

A. 価格履歴グラフ
   - 過去90日間の価格推移
   - ECサイト別の価格変動
   - セール期間のハイライト

B. スマート価格予測（ML）
   - 季節パターン検出
   - 「来週セール予測」通知
   - 過去のセール周期分析

C. 価格アラート
   - 目標価格設定
   - 「過去最安値」到達通知
   - 「平均より20%安い」通知

【訴求メッセージ】
「AIは今の価格しか知りません。Suptiaは最適な買い時を教えます」

【優先度】高（Pro会員の核機能）
【工数】2週間
【担当フェーズ】Phase 2

------------------------------------------------------------------------------
4.4 エビデンス透明性の強調
------------------------------------------------------------------------------

【差別化ポイント】
AI検索は「なぜその商品を推薦したか」の根拠が不透明。
Suptiaは5つの柱（価格・成分量・コスパ・エビデンス・安全性）で
100%根拠を明示する。

【実装内容】

A. 推薦根拠の可視化
   - 「なぜこの商品が1位か」を5つの柱で説明
   - 各柱のスコア内訳を展開表示
   - 参考文献へのワンクリックアクセス

B. 比較レポート機能
   - 選択した商品の詳細比較PDF
   - 5つの柱の比較表
   - 根拠付きの推薦理由

C. 「AI検索との違い」ページ
   - Suptiaの透明性を強調するランディングページ
   - 「AI検索の限界」と「Suptiaの強み」を対比
   - SEO・コンバージョン向上

【訴求メッセージ】
「AIは答えを出しますが、根拠を説明しません。Suptiaは全ての推薦に理由があります」

【優先度】中
【工数】1週間
【担当フェーズ】Phase 2


================================================================================
                    5. 戦略C: AI検索との直接連携
================================================================================

------------------------------------------------------------------------------
5.1 ChatGPT Plugin / GPT Actions対応
------------------------------------------------------------------------------

【概要】
ChatGPTのPlugin機能やGPT Actionsを通じて、Suptiaのデータを
ChatGPT内で直接利用できるようにする。

【メリット】
- ChatGPTユーザーがSuptiaのデータを自然に利用
- 「Suptiaで調べて」と言えば詳細情報を取得
- アフィリエイトリンク経由の収益

【実装内容】

A. OpenAPI仕様書の作成
   - 成分検索エンドポイント
   - 商品検索エンドポイント
   - 価格比較エンドポイント
   - 安全性チェックエンドポイント

B. GPT Actions設定
   - 自然言語クエリの解釈
   - Suptia APIへのリクエスト
   - 結果のフォーマット

【優先度】中
【工数】2週間
【担当フェーズ】Phase 3

------------------------------------------------------------------------------
5.2 Perplexity / Google SGE対応
------------------------------------------------------------------------------

【概要】
Perplexity SearchやGoogle SGEがSuptiaを情報源として
引用するように最適化する。

【実装内容】

A. 引用されやすいコンテンツ構造
   - 「〜とは」で始まる定義文
   - 箇条書きでの要点整理
   - 数値データの明確な提示

B. E-E-A-T強化
   - 著者情報の明示（専門家監修）
   - 参考文献の充実
   - 更新日時の明確化

C. ファクトチェック可能な記述
   - 主張には必ずソースを付与
   - 「〜によると」の形式で引用

【優先度】中
【工数】継続的な改善
【担当フェーズ】Phase 2-3

================================================================================
                        8. リスクと対策
================================================================================

【リスク1】AI検索が全ての購買行動を奪う
対策:
- AI検索にはない「パーソナライズ」「継続サポート」で差別化
- AI検索の情報源としてのポジションを確保
- B2B向けデータライセンスで収益多角化

【リスク2】AI検索がSuptiaのデータを無断利用
対策:
- 利用規約の明確化（llms.txtで記載）
- 引用要件の明示
- 必要に応じて法的対応

【リスク3】構造化データがAI検索に活用されない
対策:
- 継続的なSEO/構造化データの改善
- AI検索エンジンの仕様変更に追従
- 複数のAI検索エンジンに対応（分散リスク）

【リスク4】ユーザーがAI検索の情報を鵜呑みにする
対策:
- 「AI検索の限界」を啓蒙するコンテンツ
- 薬機法・安全性の重要性を訴求
- 「根拠のある情報」の価値を伝える


================================================================================
                            9. まとめ
================================================================================

【Suptiaの戦略方針】

1. AI検索を敵視せず、「共存パートナー」として捉える

2. AI検索に「選ばれるデータソース」になる
   - llms.txt、構造化データ、APIの整備
   - 信頼性の高いエビデンスデータの提供

3. AI検索では提供できない価値で差別化
   - 薬機法コンプライアンス
   - パーソナライズド安全性判定
   - 価格履歴・買い時予測
   - 根拠の100%明示

4. AI検索と直接連携してシナジーを生む
   - ChatGPT Plugin / GPT Actions
   - API公開によるデータ提供

【キャッチフレーズ案】

「AIが答えを出す時代。Suptiaはその根拠を示す。」

「AI検索は便利。でも、あなたの身体のことはSuptiaに聞いて。」

「AIは一般論。Suptiaはあなた専用。」
