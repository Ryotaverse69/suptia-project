# note記事作成スキル

サプティアのnoteアカウントで公開する記事を作成するためのガイド。noteの高ドメインパワー（96点）を活用してSEO流入を獲得し、suptia.comへ誘導する。

## ブランド表記ルール

| 表記                 | 用途                                          |
| -------------------- | --------------------------------------------- |
| サプティア           | 基本表記（本文、説明文、ページタイトルなど）  |
| サプティア（Suptia） | 初出・正式表記                                |
| Suptia               | URL、ハッシュタグ、リンクテキスト、英語圏向け |

**例：**

- ⭕ 「サプティアでは5つの軸で評価しています」
- ⭕ 「詳細は[サプティア](https://suptia.com)で確認できます」
- ⭕ `#Suptia`（ハッシュタグは英語OK）

## 目的

```
noteで検索上位獲得 → 記事を読む → suptia.comへ誘導 → 被リンク効果 + トラフィック増加
```

## 記事出力形式

**重要**: 記事は必ずマークダウン形式で出力してください。noteに直接コピー&ペーストできる形式で提供します。

### 出力構成（3ブロックに分離）

記事出力時は以下の3ブロックを**別々のコードブロック**で出力し、個別にコピーできるようにする：

1. **記事本文**（マークダウン）
2. **ハッシュタグ**（別ブロック）
3. **画像生成プロンプト**（別ブロック）

### Amazonリンクの出力形式

Amazonリンクは**商品カード形式**で出力する。URLを単独行で記載すると、noteが自動で商品カード（画像・価格・商品名付き）に変換する。

```markdown
❌ NG（テキストリンク）:
▶ [Amazonで見る](https://www.amazon.co.jp/dp/B01FHREIKI?tag=suptia6902-22)

⭕ OK（商品カード形式）:
▶ Amazonで見る
https://www.amazon.co.jp/dp/B01FHREIKI?tag=suptia6902-22
```

### ハッシュタグの出力

ハッシュタグは記事本文には含めず、**別のコードブロック**で出力する。

```markdown
<!-- 記事本文のコードブロック終了後 -->

---

**ハッシュタグ（個別コピー用）:**
```

#サプリメント #ビタミンC #コスパ #健康 #美容


## 記事カテゴリ（3種類）

| カテゴリ           | 割合 | 目的           | 文字数         |
| ------------------ | ---- | -------------- | -------------- |
| **SEO記事**        | 60%  | 検索流入獲得   | 2,500〜3,500字 |
| **ブランディング** | 30%  | 信頼構築・認知 | 1,500〜2,500字 |

## キーワード戦略

### 狙うべきキーワード（ロングテール）

| カテゴリ     | 例                                  | 理由             |
| ------------ | ----------------------------------- | ---------------- |

### 避けるキーワード

- 単一ワード（競合強すぎ）
- 「サプリ おすすめ」（マイベスト等が強い）
- 医療系ワード（薬機法リスク）

## タイトルの付け方

### 基本ルール

- **30文字前後**（検索結果で途切れない）
- **数字を入れる**（「5選」「3つの理由」）
- **ベネフィットを明確に**
- **年号を入れる**（「2026年」）

### テンプレート

```
【2026年】〇〇サプリ コスパ比較｜476商品から厳選5選
〇〇サプリの選び方｜失敗しない3つのポイント
なぜ〇〇サプリは効果がないと言われるのか｜科学的に検証
40代からの〇〇サプリ｜目的別おすすめ完全ガイド
```

### NGパターン

- 長すぎる（40文字以上）
- 抽象的（「おすすめサプリまとめ」）
- 煽りすぎ（「絶対に〇〇」「衝撃の事実」）

## 記事フォーマットルール

### 禁止事項

| 記法             | 禁止理由             | 代替案             |
| ---------------- | -------------------- | ------------------ |
| `**太字**`       | noteでは視認性が悪い | 見出し（##）で強調 |
| `***斜体太字***` | 可読性低下           | 使用しない         |
| 絵文字の多用     | うるさい印象         | 1見出しに1個まで   |

### 推奨フォーマット

- 見出し: `##` `###` で階層化
- 強調: 「」で囲む、または見出しにする
- リスト: `・` または `-` を使用
- 引用: `>` でポイントを目立たせる
- 区切り: `---` で視覚的に分離

### 例

```markdown
❌ NG: これは**重要な**ポイントです。
⭕ OK: これは「重要なポイント」です。
⭕ OK:

### 重要なポイント

これは覚えておくべき内容です。
```

## 重複防止ルール

**重要**: 記事作成前に必ず [references/published-articles.md](references/published-articles.md) を確認する。

### チェック手順

1. `published-articles.md` を読み込む
2. 使用済みキーワードを確認
3. 新しい切り口かどうか判断
4. 記事公開後、ファイルを更新

### 重複の判断基準

| 状況                  | 判断    |
| --------------------- | ------- |
| 同じ成分 × 同じ切り口 | ❌ 重複 |
| 同じ成分 × 違う切り口 | ✅ OK   |
| 違う成分 × 同じ切り口 | ✅ OK   |

例：

- 「ビタミンD コスパ比較」公開済み → 「ビタミンD 冬の摂取量」は ✅ OK
- 「ビタミンD コスパ比較」公開済み → 「ビタミンC コスパ比較」は ✅ OK
- 「ビタミンD コスパ比較」公開済み → 「ビタミンD サプリ おすすめ」は ❌ 重複

## 既存記事のリライト

サプティアの比較記事をnote向けにリライトする場合：

1. **元記事を確認**: `/articles/[ingredient]-comparison`
2. **要約版を作成**: 3,000字程度に凝縮
3. **note向けに調整**:
   - 冒頭で価値を伝える
   - 結論を先に出す
   - リンクをnote記法に変換
4. **CTAを追加**: 「詳細はサプティアで」

## 画像生成プロンプト

記事作成時に以下の画像プロンプトも同時に出力する。
画像は `/admin/note-images` でGemini 3 Pro Image Previewを使用して生成。

**重要な出力フォー��ット**:

1. まず完成した記事本文をマークダウン形式で全文出力
2. その後、セパレーター（`---`）で区切る
3. 最後に画像生成プロンプトを3つ出力

**画像生成プロンプトについて**:

- これらのプロンプトは画像生成AI（Gemini）に渡すためのテキストです
- 「含めないもの」はネガティブプロンプト（生成してほしくない要素）として必須
- 記事本文には含めず、画像生成時のみ使用します

### 記事内の画像配置

これらの箇所に画像を配置してください：
